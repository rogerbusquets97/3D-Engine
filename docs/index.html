Introduction

Roger's Engine is a 3D engine meant for game development, developed by Roger Busquets and Sergi Parra from CITM in BarcelonaTech as an assignment for a Game Engines class.

  • Roger Busquets

    Worked on the audio system, Devil implementation to load textures as well as space
    partitioning and the GUI.

  • Sergi Parra

    Worked on the GameObjects system, including hierarchy and editing, as well as file loading and scene serialization, implementing the Assimp library.


The engine's subsystems

Geometry loading

The engine can load FBX files and read the geometry in them, moving them to its own, more efficient format, and, when loading them, loading them into the GPU to avoid useless CPU usage

Texture loading

The engine can load texture files in most formats and apply them to any geometry that has been loaded. It will also auto-load textures specified in an FBX file

Frustum Culling/Face Culling

The engine uses frustum culling and face culling to avoid rendering useless graphics when in use

Gameobjects

Every geometry loaded will be loaded in a gameobject with its own properties that can be modified, all of them with their hierarchy and dependency, changing a parent will change their childs

Space partitioning

The engine partitions space to avoid useless collision detections, using QuadTrees it efficiently tests collisions

Time management

The engine can pause and play the actions inside the engine, including audio


Audio subsytem

Roger's Engine works with the Wwise library fully, which adds 3D audio, and can load whole soundbacks and assign them to sound events, it has listeners, which, as the name suggests, listens for audio, and sources, which emit it, sources can access the events in soundbanks and reproduce different sounds. Wwise has also its own editor that lets you work on your audio files. If a file is assigned as 3D inside this editor, the sound will play according to the position in 3D space where it is being emitted, and will move if this source moves.
